# Issue Report Spec

분석 에이전트(보안, 성능, 커버리지 등)가 발견한 이슈를 보고할 때 사용하는 통일 형식입니다.

## 구조

리포트는 **Major**와 **Minor** 두 섹션으로 구성됩니다.

### Major

반드시 조치가 필요한 항목. 방치하면 장애, 보안 침해, 데이터 유실, 또는 스펙 미충족으로 이어질 수 있는 이슈.

### Minor

개선하면 좋은 항목. 현재 동작에는 문제 없으나, 품질·유지보수·성능 측면에서 권장되는 이슈.

## 마크다운 출력 형식

```markdown
### Major

#### [항목 제목] (`파일경로:라인번호`)
[이슈 설명과 개선 방안을 자유롭게 서술]

#### [항목 제목] (`파일경로:라인번호`)
[이슈 설명과 개선 방안을 자유롭게 서술]

### Minor

#### [항목 제목] (`파일경로:라인번호`)
[이슈 설명과 개선 방안을 자유롭게 서술]
```

## 작성 가이드

- 각 항목에 **위치(파일:라인)**를 반드시 포함
- 설명에는 현재 상태와 개선 방안을 함께 서술
- 항목 분류(Major/Minor)는 에이전트가 분석 관점에 따라 자유롭게 판단
- 항목이 없는 섹션은 "없음"으로 표기
- 리포트 마지막에 **Major N건 / Minor N건** 통계 한 줄 포함
