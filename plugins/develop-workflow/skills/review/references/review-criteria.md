# 상세 리뷰 평가 기준

## 기본 평가 기준 (각 20점, 총 100점)

### 1. 완성도 (Completeness) - 20점

| 점수 | 기준 |
|------|------|
| 18-20 | 모든 필요 정보 완비, 상세함 |
| 15-17 | 대부분 완비, 일부 추가 필요 |
| 12-14 | 기본 정보는 있으나 상세도 부족 |
| 0-11 | 중요 정보 누락 다수 |

### 2. 일관성 (Consistency) - 20점

| 점수 | 기준 |
|------|------|
| 18-20 | 완벽한 일관성 |
| 15-17 | 사소한 불일치만 존재 |
| 12-14 | 몇 가지 일관성 문제 |
| 0-11 | 심각한 모순 존재 |

### 3. 명확성 (Clarity) - 20점

| 점수 | 기준 |
|------|------|
| 18-20 | 매우 명확하고 이해하기 쉬움 |
| 15-17 | 대체로 명확, 일부 개선 여지 |
| 12-14 | 이해 가능하나 애매한 부분 존재 |
| 0-11 | 불명확하고 이해하기 어려움 |

### 4. 실현 가능성 (Feasibility) - 20점

| 점수 | 기준 |
|------|------|
| 18-20 | 매우 현실적이고 실행 가능 |
| 15-17 | 대체로 실행 가능, 일부 조정 필요 |
| 12-14 | 실행 가능하나 리스크 존재 |
| 0-11 | 비현실적이거나 실행 어려움 |

### 5. 품질 (Quality) - 20점

| 점수 | 기준 |
|------|------|
| 18-20 | 탁월한 품질, 모범 사례 |
| 15-17 | 높은 품질, 일부 개선 가능 |
| 12-14 | 양호한 품질, 개선 여지 존재 |
| 0-11 | 품질 미흡 |

## 관점별 특화 기준

### 엔지니어 관점
- **기술적 타당성**: 스택 적절성, 아키텍처 합리성, 안티패턴 회피
- **확장성**: 미래 성장, 모듈화, 성능
- **유지보수성**: 가독성, 복잡도, 테스트 가능성
- **기술 부채**: 트레이드오프, 리팩토링, 의존성

### 보안 관점
- **인증/권한**: 접근 제어, 세션 관리
- **입력 검증**: SQL Injection, XSS, CSRF
- **데이터 보호**: 암호화, 민감 데이터 처리

### 독자 관점
- **가독성**: 문장 명확성, 전문 용어 적절성
- **흥미도**: 후킹, 클리프행어
- **몰입감**: 감정적 연결, 캐릭터 공감
- **페이싱**: 전개 속도, 긴장/이완 균형

### 비즈니스 관점
- **시장 가치**: 타겟 시장, 경쟁 우위
- **실행 가능성**: 리소스, 타임라인
- **ROI**: 비용 대비 효과
- **리스크**: 식별, 완화 전략

## 가중치 조정

| 관점 | 완성도 | 일관성 | 명확성 | 실현 가능성 | 품질 |
|------|--------|--------|--------|------------|------|
| 기술 문서 | 25% | 20% | 20% | 25% | 10% |
| 소설/콘텐츠 | 15% | 20% | 15% | 10% | 40% |
| 비즈니스 | 20% | 15% | 20% | 30% | 15% |

## 점수 해석

| 점수 | 등급 | 해석 |
|------|------|------|
| 90-100 | Excellent | 바로 적용 가능 |
| 75-89 | Good | 경미한 수정 후 적용 |
| 60-74 | Fair | 상당한 개선 필요 |
| 0-59 | Needs Work | 재설계 권장 |
