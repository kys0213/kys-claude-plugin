# Planning Plugin

요구사항 기반 상위 레벨 아키텍처 설계 플러그인입니다. Claude, OpenAI Codex, Google Gemini 3개 LLM을 병렬로 호출하여 다양한 관점의 설계 의견을 수집하고, ASCII 다이어그램으로 도식화합니다.

## 특징

- **요구사항 수집**: Phase 0에서 기능/비기능 요구사항, 제약조건 구체화
- **멀티 LLM 설계**: 3개 LLM이 동일한 요구사항에 대해 각자의 관점으로 분석
- **ASCII 도식화**: 컴포넌트 다이어그램과 데이터 흐름도를 ASCII로 시각화
- **통합 요약**: 3개 의견을 하나의 종합 리포트로 정리

## 워크플로우

```
Phase 0: 요구사항 수집
    │   - 기능/비기능 요구사항
    │   - 제약조건 (기술 스택, 팀 규모)
    │   - 우선순위 (Must/Should/Could)
    ▼
Phase 1: 아키텍처 설계 (3개 LLM 병렬)
    ├── Claude ─┐
    ├── Codex  ─┼─→ 취합
    └── Gemini ─┘
    │
    ▼
Phase 2: 통합 및 도식화
    └── ASCII 다이어그램 생성
```

## 설치

### 사전 요구사항

- Claude Code CLI
- OpenAI Codex CLI (`codex` 명령어)
- Google Gemini CLI (`gemini` 명령어)

### 마켓플레이스에서 설치

```bash
claude install kys0213/kys-claude-plugin/planning
```

## 사용법

```bash
# 새 시스템 설계
/outline "실시간 채팅 시스템을 설계해줘"

# 제약조건 포함
/outline "React + Node.js로 e-commerce 플랫폼 설계. 동시 사용자 10만명"

# 기존 시스템 확장
/outline "현재 모놀리스를 마이크로서비스로 전환하는 설계"
```

## 컴포넌트

### 커맨드

| 이름 | 설명 |
|------|------|
| `/outline` | 요구사항 기반 상위 레벨 설계 수행 |

### 에이전트

| 이름 | 설명 |
|------|------|
| `outline-claude` | Claude를 사용한 아키텍처 설계 |
| `outline-codex` | OpenAI Codex를 사용한 아키텍처 설계 |
| `outline-gemini` | Google Gemini를 사용한 아키텍처 설계 |

### 스킬

| 이름 | 설명 |
|------|------|
| `high-level-design` | 상위 레벨 설계 원칙 및 가이드라인 |

## 기존 Plan Mode와의 차이점

| 항목 | 기존 Plan Mode | /outline |
|------|----------------|----------|
| 시작점 | 코드 분석 | 요구사항 수집 |
| 초점 | 구체적인 코드 수정 | 아키텍처/기술선택 |
| 출력 | 파일별 수정 계획 | ASCII 다이어그램 + 설계서 |
| 관점 | 단일 (Claude) | 다중 (3개 LLM) |
| 단계 | How (어떻게) | What & Why (무엇을, 왜) |
