# Flow 8: 스펙 완료 판정

### 시나리오

Claw가 스펙의 모든 요구사항이 충족되었다고 판단한다.

### 완료 조건

```
AND 조건 (모두 충족해야 Complete):
  1. 모든 linked_issues가 done 상태
  2. gap detection에서 새로운 gap이 발견되지 않음
  3. Acceptance Criteria가 검증 가능한 경우: 테스트 통과
```

### 완료 플로우

```
  Claw: 완료 조건 충족 판단
         │
         ▼
  ┌──────────────────────────────────────┐
  │  1. Acceptance Criteria 자동 검증      │
  │     → daemon이 대상 레포에서 테스트 실행│
  │       (spec.test_environment 명령 사용)│
  │     → Claw가 결과 해석 + 판정         │
  └──────────┬───────────────────────────┘
             │
        전체 통과?
        ├── NO → 실패한 criteria를 이슈로 등록
        │         → 자율 루프 계속
        │
        └── YES
             │
             ▼
  ┌──────────────────────────────────────┐
  │  2. HITL: 최종 확인 요청              │
  │     → GitHub 이슈 코멘트:            │
  │       "스펙의 모든 요구사항이          │
  │        충족되었습니다.                │
  │        [결과 리포트]                  │
  │        확인 후 스펙을 완료해주세요."   │
  │                                      │
  │     응답 방법 (Flow 5 참조):          │
  │     - CLI: autodev hitl respond ...  │
  │     - GitHub: HITL 코멘트에 답글      │
  └──────────┬───────────────────────────┘
             │
         사용자 확인
        ├── 추가 피드백 → Flow 7로
        │
        └── 확인
             │
             ▼
  ┌──────────────────────────────────────┐
  │  3. spec.status = Completed          │
  │  4. 완료 리포트 생성                  │
  │  5. (옵션) PR들 자동 merge           │
  └──────────────────────────────────────┘
```
