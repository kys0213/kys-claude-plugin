# Conflict Resolver

머지 시 발생하는 충돌을 분석하고 자동으로 해결합니다.

## 해결 프로세스

### 1. 충돌 파일 파악

```bash
git diff --name-only --diff-filter=U
```

### 2. 충돌 분석

각 충돌 파일에 대해:
- `<<<<<<<` ~ `=======` ~ `>>>>>>>` 마커 파악
- 양측(ours, theirs)의 변경 의도 분석
- base 버전과 비교하여 각 변경의 목적 이해

### 3. 해결 전략

| 상황 | 전략 |
|------|------|
| 서로 다른 부분 수정 | 양측 모두 반영 |
| 동일 부분 수정, 호환 가능 | 두 변경 통합 |
| 동일 부분 수정, 상충 | 최신 의도 우선 + 기능 보존 |
| 구조적 변경 (파일 이동 등) | 새 구조에 맞게 재적용 |

### 4. 해결 적용

Edit 도구로 충돌 마커를 제거하고 올바른 코드로 교체합니다.

### 5. 검증

```bash
# 빌드 확인
# 테스트 실행 (가능한 경우)
git add <resolved_files>
```

## 주의사항

- 확신이 없는 충돌은 해결하지 않고 보고합니다
- 해결 후 반드시 빌드/테스트 통과 여부를 확인합니다
