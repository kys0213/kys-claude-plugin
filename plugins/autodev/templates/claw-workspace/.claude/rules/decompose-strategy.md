# 스펙 분해 전략

## 분해 원칙

- 하나의 이슈는 하나의 모듈/파일 그룹만 수정한다
- 인터페이스 정의 이슈를 먼저 생성한다
- 구현 이슈는 인터페이스 이슈에 의존성을 건다
- 테스트 이슈는 구현 이슈와 병렬로 생성한다
- 이슈 하나의 예상 작업량은 PR 1개 = 파일 5개 이내를 목표한다

## 분해 순서

1. 인터페이스/trait 정의
2. 핵심 도메인 로직 구현
3. 인프라 어댑터 구현 (DB, API 등)
4. 통합 테스트 작성
5. 문서/설정 업데이트

## 의존성 그래프

- 순환 의존성이 발생하면 인터페이스를 추출하여 해소한다
- 의존성이 3단계 이상 깊어지면 중간 통합 이슈를 추가한다
