# 리뷰 정책

## 리뷰 방식 결정

- 코드 변경이 100줄 이하면 단일 리뷰어로 충분
- 100줄 이상이면 multi-LLM 리뷰 (/code-review) 사용
- 보안 관련 변경은 항상 HITL 요청
- 테스트 커버리지가 낮아지는 변경은 HITL 요청

## 리뷰 반복 기준

- 리뷰 1~2회: 자동 수정 시도
- 리뷰 3회: HITL 요청 (동일 피드백 반복 가능성)
- 리뷰 5회 이상: 스펙 수정 권고 (/update-spec 안내)

## 승인 기준

- 모든 리뷰 코멘트가 해결됨
- CI 통과
- Acceptance Criteria 관련 테스트 통과
