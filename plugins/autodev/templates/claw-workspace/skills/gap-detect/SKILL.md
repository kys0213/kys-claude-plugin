# 스펙-코드 갭 탐지

스펙의 요구사항/아키텍처와 실제 코드 상태를 대조하여 누락, 불일치, 미완성 영역을 탐지합니다.

## 탐지 대상

### 1. 구조 갭

스펙에 정의된 모듈/파일이 코드베이스에 존재하는지 확인합니다:
- 스펙의 아키텍처 섹션에 명시된 파일/모듈이 생성되었는지
- 정의된 인터페이스/trait이 구현되었는지
- 필요한 의존성이 추가되었는지

### 2. 기능 갭

스펙의 요구사항이 코드에 반영되었는지 확인합니다:
- API 엔드포인트가 정의되었는지
- 비즈니스 로직이 구현되었는지
- 에러 핸들링이 포함되었는지

### 3. 테스트 갭

Acceptance Criteria에 대응하는 테스트가 존재하는지 확인합니다:
- 각 criteria에 대한 테스트 케이스 존재 여부
- 테스트가 실제로 통과하는지 (실행 가능한 경우)

### 4. 컨벤션 갭

기술 스택/컨벤션 섹션과 실제 코드의 일치 여부:
- 지정된 프레임워크/라이브러리 사용 여부
- 코딩 컨벤션 준수 여부

## 탐지 절차

1. **스펙 로드**: `autodev spec show <id> --json`으로 스펙 내용 조회
2. **코드 스캔**: 대상 레포의 코드베이스를 Glob/Grep으로 탐색
3. **대조**: 스펙의 각 항목과 코드 상태를 매칭
4. **갭 리포트**: 발견된 갭을 유형별로 정리

## 갭 발견 시 처리

- **구조/기능 갭**: 새 이슈 생성 (autodev:analyze 라벨)
- **테스트 갭**: 테스트 추가 이슈 생성
- **심각한 갭** (아키텍처 불일치): HITL 요청 + /update-spec 권고

## 트리거

- `gap-detection` cron이 주기적으로 실행 (기본: 1시간)
- git 변경이 있을 때만 실행 (guard)
- 수동: `autodev spec evaluate <id>`
