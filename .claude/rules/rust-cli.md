# CLI 레이어 컨벤션 (사용자 인터페이스)

> CLI는 시스템의 가장 바깥 경계다. 내부 리팩토링에 테스트가 깨지지 않아야 한다.

## 블랙박스 TDD 필수

- 테스트는 내부 구현이 아닌 외부 동작(입력 → 출력)을 기준으로 작성한다.
- 외부 의존성(DB, API, 파일시스템)은 mock을 주입하여 격리한다.
- core/service/infra 내부가 리팩토링되어도 CLI 테스트는 깨지지 않아야 한다.

## 비즈니스 로직을 CLI에 두지 마라

- CLI는 인자 파싱 → service 호출 → 결과 포매팅만 담당한다.
- 조건 분기, 상태 변환 등 로직이 CLI에 있으면 service로 이동한다.
