# Service 레이어 컨벤션 (비즈니스 로직)

> core가 제공하는 타입과 메서드를 올바르게 소비하는 것이 이 레이어의 책임이다.

## match는 exhaustive하게 작성하라

- `_ =>` wildcard 대신 모든 variant를 명시적으로 나열한다.
- 새 variant가 추가될 때 컴파일 에러로 누락을 알 수 있다.

## core가 제공하는 전이 메서드를 사용하라

- 도메인 객체의 pub 필드를 직접 변경하여 상태를 전이하지 마라.
- core가 `transition_to_*()`, `set_*()` 등 전용 메서드를 제공하면 그것을 사용한다.

## 변환 목적의 중간 타입을 만들지 마라

- 임시 struct가 대상 타입과 필드가 동일하면 중복이다.
- 대상 타입을 직접 사용하여 변환 경로를 단축한다.
