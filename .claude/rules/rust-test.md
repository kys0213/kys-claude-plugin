# Test 컨벤션

> Rust는 `#[cfg(test)] mod tests`(같은 파일, 화이트박스)와 `tests/` 디렉토리(블랙박스)를 구분한다.
> 이 규칙은 `tests/` 디렉토리의 통합 테스트에 적용된다.
> 같은 파일 내 `mod tests`는 해당 레이어 컨벤션(core/service/infra/cli)을 따른다.

## 통합 테스트는 pub API만 사용하라

- crate의 public interface만으로 테스트를 작성한다.
- 내부 리팩토링에 테스트가 깨지면 테스트가 구현에 결합된 것이다.

## 팩토리는 공유 모듈로 통합하라

- 동일한 테스트 데이터 생성 코드가 3곳 이상 반복되면 `#[cfg(test)] pub mod testing`으로 추출한다.
- 프로덕션 바이너리에 포함되지 않으면서 모든 테스트 모듈에서 재사용 가능하다.

## 동작을 검증하라

- 데이터 구조가 올바르게 생성되었는지가 아니라, 비즈니스 동작이 올바른지를 검증한다.
- 상태 전이, 타입 라우팅, 리소스 격리 등 새로 도입된 동작에 대한 테스트가 누락되지 않았는지 확인한다.
